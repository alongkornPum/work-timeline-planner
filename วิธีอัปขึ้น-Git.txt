==================================================================
  วิธีอัปโค้ดขึ้น Git + Deploy เว็บจริง (Work Timeline Planner)
==================================================================

โค้ดเดินทาง 3 ที่:
   เครื่องคุณ  --push-->  GitHub  --auto-->  Vercel (เว็บจริง)
   (แก้โค้ด)             (เก็บโค้ด)          (build + deploy เอง)

หน้าที่คุณ = ส่งโค้ดจากเครื่อง -> GitHub ที่เหลือ Vercel ทำเอง

Repo ของโปรเจกต์นี้:
   https://github.com/alongkornPum/work-timeline-planner


------------------------------------------------------------------
  ★ ทุกครั้งที่แก้โค้ดเสร็จ ใช้ 3 คำสั่งนี้
------------------------------------------------------------------

   git add -A
   git commit -m "ข้อความบอกว่าทำอะไร"
   git push

ความหมายแต่ละคำสั่ง:
   git add -A          = เลือกไฟล์ที่แก้ทั้งหมด เตรียมบันทึก (หยิบของใส่กล่อง)
   git commit -m "..." = บันทึกเป็นเวอร์ชันใหม่ในเครื่อง (ปิดกล่อง + ติดป้าย)
   git push            = ส่งขึ้น GitHub (ส่งกล่องออกไป)

ตัวอย่างจริง:
   git add -A
   git commit -m "เพิ่มช่องชื่อ-นามสกุลในหน้าสมัคร"
   git push


------------------------------------------------------------------
  ★ คำสั่งช่วยตรวจสอบ (ใช้ก่อน/หลัง push ได้)
------------------------------------------------------------------

   git status              = ดูว่ามีไฟล์ไหนแก้ไปบ้าง / ยัง commit หรือยัง
   git log --oneline -5    = ดูประวัติ 5 commit ล่าสุด
   git branch --show-current = ดูว่าอยู่ branch ไหน (ต้องเป็น main)


------------------------------------------------------------------
  ★ ทางลัด: ทำผ่านปุ่มใน VS Code (ไม่ต้องพิมพ์คำสั่ง)
------------------------------------------------------------------

   1. กดไอคอน Source Control แถบซ้าย (รูปกิ่งไม้) หรือกด Ctrl+Shift+G
   2. เห็นรายการไฟล์ที่แก้ -> กดเครื่องหมาย +  ที่ไฟล์ (= git add)
   3. พิมพ์ข้อความในช่องด้านบน -> กดปุ่ม  ✓ Commit
   4. กดปุ่ม  Sync Changes / Push   (= git push)


------------------------------------------------------------------
  ★ หลัง push แล้วเกิดอะไรขึ้น
------------------------------------------------------------------

   - Vercel จับการ push อัตโนมัติ -> build ~1 นาที -> เว็บจริงอัปเดต
   - ดูสถานะที่ vercel.com -> โปรเจกต์ -> แท็บ Deployments
        ✅ Ready    = เสร็จแล้ว
        ⏳ Building = กำลัง build
        ❌ Error    = build ไม่ผ่าน (กดเข้าไปดู log)
   - ลิงก์ .vercel.app เดิมไม่เปลี่ยน
   - เลข version (v1.0.x) ใต้ icon จะ +1 อัตโนมัติทุก commit ที่ deploy


------------------------------------------------------------------
  ★ กรณีพิเศษ: แก้ Environment Variables (เช่น คีย์ Supabase)
------------------------------------------------------------------

   การ push เฉย ๆ ไม่พอ! เพราะค่า VITE_* ถูกฝังตอน build
   ต้อง: Vercel -> Settings -> Environment Variables -> แก้ -> Save
         แล้วไปแท็บ Deployments -> กด ... ที่ deploy ล่าสุด -> Redeploy


------------------------------------------------------------------
  ★ ปัญหาที่พบบ่อย + วิธีแก้
------------------------------------------------------------------

1) push แล้วถาม username/password หรือเด้งหน้าต่าง login
   -> ครั้งแรกต้อง login GitHub ผ่านเบราว์เซอร์ กดยืนยันได้เลย (ครั้งเดียว)

2) error: "Author identity unknown"
   -> ตั้งชื่อ/อีเมลให้ git (ทำครั้งเดียว):
        git config user.email "dpu_claude006@dpu.ac.th"
        git config user.name  "DPU Claude"

3) error: "failed to push ... rejected" (มีคนแก้บน GitHub ก่อน)
   -> ดึงของใหม่ลงมาก่อนแล้วค่อย push:
        git pull --no-rebase
        git push

4) เผลอ commit แต่ยังไม่ push
   -> push ทีหลังได้เลย:  git push
      (commit อยู่ในเครื่องก่อน จนกว่าจะ push)

5) อยากรู้ว่าไฟล์ .env หลุดขึ้น GitHub ไหม
   -> ไม่หลุด! .env อยู่ใน .gitignore แล้ว (คีย์ปลอดภัย)
